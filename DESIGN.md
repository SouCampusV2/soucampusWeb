# Дизайн-система SouCampus builds

Справочник по вёрстке/стилю проекта. `CLAUDE.md` — план и статус проекта; сюда вынесено всё, что касается конкретно дизайна, чтобы не перегружать план. Обновлять этот файл при любой предметной правке дизайн-системы (новый цвет, новое правило вёрстки, новый переиспользуемый компонент).

Референс по общему духу — Wise (wise.com) и отчасти Revolut: плоский дизайн, уверенная типографика, много воздуха, крупные скругления, светлый фон, несколько ярких акцентных цветов вместо одного.

## Цвета

Три акцентных цвета вместо одного, по образцу того, как Wise/Revolut используют несколько ярких цветов внутри одной нейтральной базы:

- **Orange — primary.** Главный интерактивный цвет: все primary-кнопки, основной CTA, логотип. Один и тот же цвет для всех "сделай действие" элементов сайта.
- **Blue — secondary accent.** Категоризация/информационные акценты (например один из тарифов в `PricingSection`).
- **Lime — tertiary accent.** Позитивные/бонусные акценты, "продвинутый" вариант где нужен третий цвет.
- Правило: **кнопки всегда orange** — заливкой (primary), обводкой (secondary) или подчёркнутым текстом (tertiary), см. ниже — blue/lime не красят кнопки, только бейджи, обводки, декоративные элементы, чекмарки. Так множественность цветов не превращается в хаос из разноцветных CTA. (Исключение — `tertiary` через `colorClassName`: синие текст-ссылки на `/contact`.)
- Базовый уровень оттенка — `400` (мягкий, пастельный, не кислотный), но по месту используются и другие шаги шкалы — конкретные секции могут брать `300`/`350`/`500`/`650` и т.п. под конкретную задачу контраста. Не следует считать, что "везде должен быть ровно -400" — важен принцип (мягкие, не кислотные тона), а не фиксированный номер.
- **Нейтральная палитра** — `zinc` (не `gray`/`slate`) для текста, бордеров, фонов карточек. Заголовки — `zinc-950`, тело текста — `zinc-600`/`zinc-700`, второстепенный текст — `zinc-500`.
- **"Белый" на сайте — всегда `#fbfbff`, не чистый `#ffffff`.** Там, где нужен белый фон — `bg-[#fbfbff]`, не `bg-white`. `--background` в `globals.css` уже выставлен в `#fbfbff`. Чуть тёплый белый выглядит мягче, чем ледяной чистый белый.
- **Тёмная тема — реализована (2026-07-17).** Раньше `dark:`-классы были мёртвым кодом (класс `.dark` нигде не проставлялся) — теперь `src/components/ThemeProvider.tsx` реально переключает его по клику на кнопку в футере (`ContactFooter.tsx`, иконка Sun/Moon рядом с логотипом). Выбор сохраняется в `localStorage` (`theme`), плюс блокирующий `<script>` в `<head>` (`layout.tsx`, `THEME_INIT_SCRIPT`) — проставляет класс синхронно до гидратации, чтобы не мигало светлой темой при заходе с сохранённым тёмным выбором. `@custom-variant dark (&:where(.dark, .dark *));` в `globals.css` — механизм, который делает `dark:`-классы Tailwind зависимыми именно от этого класса, не от системной темы браузера/ОС.

### Правило инверсии цветов в тёмной теме

Первый черновой проход (2026-07-17) — пользователь планирует ещё поэкспериментировать с конкретными оттенками позже, эти правила — рабочая база, не финал:

- **Белый фон (`bg-[#fbfbff]`) → `zinc-950`.** Управляется через CSS-переменную `--background` (см. `.dark { --background: #09090b; }` в `globals.css`) для `<body>`, но секции, которые перекрашивают себя явным `bg-[#fbfbff]` поверх (а не полагаются на просвечивающий фон `<body>`), берут `dark:bg-zinc-950` явно.
- **Тёмный текст (`text-zinc-950`) → `text-zinc-50`**, тело текста (`text-zinc-600`/`700`) → `text-zinc-300`/`400`, второстепенный (`text-zinc-500`) → `text-zinc-400`. Бордеры `border-zinc-200` → `border-zinc-800`.
- **Акцентные цвета (orange/lime/blue) — отражение вокруг шага 500 по шкале 50…950**: `50↔950`, `100↔900`, `150↔850`, `200↔800`, `250↔750`, `300↔700`, `350↔650`, `400↔600`, `450↔550`, `500↔500`. Применяется к пастельным поверхностям (например, карточки `ClientReviews` — `bg-orange-100` → `dark:bg-orange-950`); полноценная перекраска компонента на JS-подбираемых цветах (не через `dark:`-класс, а вычисляемых в рантайме) — см. `PageTransition.tsx`, `ACCENT_SHADES_LIGHT`/`ACCENT_SHADES_DARK`.
- **Исключение — "цветные поверхности" не инвертируются.** Кнопки (`Button.tsx`, `variant="primary"` — заливка `orange-400` + тёмный текст) и любые чипы/бейджи поверх фотографий (теги проектов, значки на карточках) не подчиняются правилу инверсии — они одинаково хорошо читаются на любом фоне сайта, поэтому не имеют `dark:`-вариантов вообще.
- **`HowItWorks.tsx` (2026-07-19) — уже не исключение, дополнен `dark:`-вариантами по стандартной формуле отражения:** фон `bg-lime-300` → `dark:bg-lime-700` (шаг 300↔700), заголовки/текст — обычная инверсия (`zinc-950`→`zinc-50`, `zinc-700`→`zinc-300`), номер шага `text-blue-650` → `dark:text-blue-350` (шаг 650↔350).
- Не все компоненты покрыты дословно — общие/переиспользуемые (`Button`, `ArrowCircle`, `Skeleton`, `Navbar`, `ContactFooter`) и весь текст на страницах (`about`, `contact`, `portfolio`, `portfolio/[slug]`, `shop`, `terms`) прошли явный проход; если где-то на новой странице/компоненте забыт `dark:`-вариант — использовать эти же правила по аналогии.
- **`/about` больше не тёмная.** Раньше timeline-секция на этой странице была осознанным тёмным исключением (`bg-zinc-950`); по просьбе пользователя (2026-07-16) переведена на общую светлую тему сайта — hero и timeline теперь используют стандартный паттерн Hero-секций (см. ниже), заголовки `zinc-950`, тело `zinc-600`, акцент на цифрах возраста — `bg-gradient-to-r from-orange-400 to-lime-500 bg-clip-text` (два primary-акцента бренда вместо одного lime). На сайте больше нет тёмных секций.

### Полная шкала оттенков на каждые 50 (orange / lime / blue)

Шкала `{orange|lime|blue}` идёт с шагом **50**, а не 100. Стоковые шаги (`50/100/200/…/950`) — родные Tailwind. Недостающие (`150/250/350/450/550/650/750/850`) объявлены в `src/app/globals.css` через **`@theme`**, одной строкой на цвет:

```css
@theme {
  --color-blue-650: #1552f1;
}
```

Этого достаточно: Tailwind сам порождает из переменной все утилиты (`bg-`/`text-`/`border-`/`ring-`/…). HEX — усреднение RGB двух соседних настоящих значений Tailwind v4, не выдуманы на глаз.

**Ограничений нет — это полноценные утилиты.** Модификаторы работают все: `dark:text-blue-350`, `hover:bg-orange-350`, `sm:text-lime-750`. Ничего обходить не нужно, произвольные значения `bg-[#hex]` ради модификатора не нужны.

> **История (2026-07-20).** Раньше эти шаги были написаны руками как обычный CSS в `@layer utilities` (72 правила), а здесь стояло, что «`@theme` не заработал» и что модификаторы к ним не приделать. Это оказалось неверно: `@theme` работает, проверено на собранном CSS. Заодно чинилась настоящая, но неочевидная поломка — рукописный `.text-blue-650` **перебивал** `dark:text-blue-50`: вариант `dark:` объявлен как `:where(.dark, .dark *)` и имеет нулевую специфичность, то есть весит столько же, сколько простой класс, а при равном весе побеждает тот, кто ниже в файле — рукописный блок шёл после сгенерированных утилит. Из-за этого тёмная тема на нестоковых шагах не применялась вообще (например, номера шагов в `HowItWorks.tsx`). После перехода на `@theme` Tailwind сам ставит варианты после базовых утилит, и порядок правильный.

## Компоненты

### Кнопки — `src/components/Button.tsx`

Все кнопки и кнопко-подобные ссылки — только через этот компонент, не собирать вручную из Tailwind-классов.

- **Primary** (`variant="primary"`) — заливка `orange-400` (hover `orange-500`), **тёмный** текст (`text-zinc-950`, не белый) — контрастнее и ближе к тому, как Wise красит свои "чипы"/CTA (яркий пастельный фон + тёмный текст).
- **Secondary** (`variant="secondary"`) — та же таблетка, что и primary (`rounded-full`, те же паддинги и высота), но **без заливки**: прозрачный фон + обводка `border-2 border-orange-500` и текст того же оранжевого (`text-orange-500`, hover `orange-600`; тёмная тема `400`→`500`). Текст не `zinc-950`, как у primary: там он тёмный, потому что лежит на оранжевой заливке, а здесь заливки нет. Для «второго действия» рядом с primary, когда нужна кнопка-пилюля, а не текст-ссылка (см. «Order a map» в `Hero`).
- **Tertiary** (`variant="tertiary"`) — без фона и без обводки, просто подчёркнутый оранжевый текст (`text-orange-500 underline decoration-2 underline-offset-4`, hover `orange-600`) — как "Send money now" у Wise/Revolut. Нет `rounded-full`/паддингов — это текстовая ссылка, не кнопка-пилюля. (Это бывший `secondary` до 2026-07-21; переименован, когда `secondary` занял вариант с обводкой. Через `colorClassName` перекрашивается в другой акцент — синие ссылки на `/contact`, lime «Read story» в `ClientReviews`.)
- **Высота — фиксированная у ВСЕХ кнопок**, любого `size` и `variant`, независимо от содержимого (текст, иконка, текст+иконка): `h-12` (48px) на мобильном (`< 640px`), `h-14` (56px) на `sm:`+ (2026-07-16: раньше было `h-14` неизменно на всех экранах — на телефоне это ощущалось слишком массивно, теперь чуть компактнее). Это на самой кнопке (`base`), не на паддинге — раньше высота задавалась вертикальным `py-*` и "плавала": кнопка с иконкой внутри (см. "Join Discord" на `/contact`) получалась выше обычной текстовой того же `size`. Паддинг теперь только горизонтальный.
- Размеры (`sm`/`md`/`lg`) отличаются только горизонтальным паддингом и размером текста: `sm` (`px-5 text-sm`), `md` (`px-7 text-sm`, дефолт), `lg` (`px-8 text-base`) — высота у всех одинаковая на любом брейкпоинте (см. выше про `h-12`/`h-14`).
- Паддинг/текст размеров (`sm`/`md`/`lg`) сами по себе не меняются по брейкпоинтам — только высота (`h-12 sm:h-14`, встроена в `base`, не в `paddingBySize`/`textBySize`). Весь остальной адаптив (что сжимается/переносится) — на уровне обёртки вокруг кнопки (`flex-col sm:flex-row` и т.п.), не самой кнопки.
- Форма — `rounded-full`. Hover — **не** увеличение (`scale`), а только смена цвета (`hover:bg-orange-500` у primary и т.п.) — `hover:scale-105` был на всех кнопках сайта, убран по просьбе пользователя (2026-07-18): теперь при наведении растёт/крутится только иконка внутри кнопки (см. паттерн `group` + `group-hover:-rotate-45` на `ArrowCircle` ниже), не сама кнопка. `active:scale-95` (нажатие/тап) — отдельная история, осталась.
- `cursor-pointer` встроен в базовый класс компонента — не убирать, иначе на `<button>`-варианте (без `href`) курсор не будет показывать кликабельность (у `<a>`/`Link`-варианта курсор и так `pointer` по умолчанию браузера).
- Если кнопка ведёт на другую страницу сайта — передавать `href` и `pageTransition` (прокидывает `data-page-transition="true"` для `PageTransition.tsx`).
- `colorClassName` — точечное переопределение цвета (primary и secondary) поверх дефолтного оранжевого, по аналогии с тем же пропом у `ArrowCircle.tsx`. Использовать только для осознанных исключений/экспериментов (см. per-card цвет кнопки в `ClientReviews.tsx`), не как обычный способ красить кнопки — по умолчанию кнопки всегда orange (см. правило выше).

### Стрелка-кружок — `src/components/ArrowCircle.tsx`

Переиспользуемая круглая иконка-стрелка. Не пересобирать SVG-path стрелки в новом месте — всегда брать этот компонент.

- `direction="left" | "right"` (дефолт `"right"`) — левая версия не отдельный path, а тот же SVG, отзеркаленный через `scaleX(-1)`.
- `variant="circle" | "bare"` (дефолт `"circle"`) — `"circle"` — иконка внутри цветного кружка (как раньше, единственный вариант до этого). `"bare"` — голая стрелка без кружка/фона, размер задаётся `className` (например `h-8 w-8`), цвет — `colorClassName`, но здесь это уже просто цвет текста (`text-zinc-950`), а не фон+hover кружка. Используется, когда иконка должна идти рядом с текстом кнопки на уровне высоты текста, а не как отдельный декоративный элемент (см. кнопку "Join Discord" на `/contact`).
- `colorClassName` — в варианте `"circle"` переопределяет фон/цвет/hover кружка целиком (дефолт `bg-zinc-950 hover:bg-zinc-800 text-white`). Для акцентных цветов сюда передаётся связка вида `bg-orange-400 hover:bg-orange-500 text-white` — та же hover-логика "на шаг темнее", что у primary-кнопки.
- `label` — необязательный текст слева от кружка, `text-zinc-950 font-semibold` (как текст primary-кнопки). Вариант "текст + стрелка" вместо голой иконки.
- `cursor-pointer` встроен в сам кружок. Hover — **не** увеличение (`scale`), а смена цвета — раньше было `hover:scale-105`, убрали по просьбе пользователя, оставили только цвет.
- Используется в `ClientReviews` (кнопки карусели, кружок `bg-orange-400 hover:bg-orange-500 text-white` — primary-акцент, белая стрелка) и в `PricingSection` (декоративная стрелка на бенто-карточках плана, цвет = акцент конкретного плана). `CountriesMarquee` использует свою комбинацию `bg-zinc-950 text-lime-300`.

### Skeleton — `src/components/Skeleton.tsx`

Плейсхолдер под асинхронный/ещё не готовый контент (фото, которых пока нет). Используется на `/about` (место под фото автора) и в аватарках `ClientReviews`.

### Футер — `src/components/ContactFooter.tsx`

Подключён в `layout.tsx`, показывается на каждой странице. Классический футер (не просто CTA-кнопка): CTA "Ready to talk about your map" убран (2026-07-17, дублировался с контентом страниц) — секция сразу начинается с самого футера.

- Колонки: бренд + короткое описание (плюс переключатель темы рядом с логотипом) → "Navigate" (те же 5 ссылок, что в `Navbar.tsx`) → "Legal" (`Terms of Service`, `/terms`) → иконки соцсетей.
- Иконки соцсетей и кнопка темы — **не** через `Button.tsx` (это не CTA, а утилитарные иконки-ссылки), тот же паттерн, что гамбургер-кнопка в `Navbar.tsx` — обычный `<button>`/`<a>` с `cursor-pointer` и `hover:`-переходом, без обводки-пилюли кроме темы (у неё `rounded-full border border-zinc-200`, чтобы визуально не потеряться среди текста логотипа).
- Иконки — Phosphor (`DiscordLogo`, `InstagramLogo`, `TiktokLogo`, `GithubLogo`, `weight="fill"`, `size={22}`), `text-zinc-500 hover:text-orange-500`.
- **Переключатель темы — рабочий (с 2026-07-17).** Кнопка (`Sun`/`Moon` иконка Phosphor) вызывает `toggleTheme()` из `useTheme()` (`ThemeProvider.tsx`), реально переключает класс `.dark` на `<html>` и сохраняет выбор в `localStorage`. См. "Тёмная тема" выше за подробностями механизма и правилами инверсии цветов.
- Копирайт-строка — отдельно под всеми колонками, с `border-t`.

### Scroll-linked кольца — `src/components/AuthorCta.tsx`

Замыкающая секция главной (перед `PricingSection`/`ContactFooter` из `layout.tsx`): концентрические кольца вокруг силуэта автора, растущие при скролле секции и слегка смещающиеся за курсором (параллакс), на обычном светлом фоне (`#fbfbff`) — без тёмного исключения. Построена по HTML/CSS/JS-референсу, который пользователь сверстал сам, переведена на React + наш стек. Ключевое отличие от референса: там был один blur-градиент-фон плюс кольца-обводки с glow; здесь каждое кольцо — заливка одним плоским шагом шкалы `lime` (`lime-100`…`lime-500`), а сам "градиент" получается наложением кругов от большего к меньшему (большой круг рисуется первым и остаётся виден только по краям — кольцом — там, где его не перекрывает следующий, меньший круг сверху). Технически для скролла — тот же приём, что и в `CountriesMarquee` (`getBoundingClientRect` + `requestAnimationFrame` + lerp к целевому значению; формула прогресса `1 - rect.bottom / (windowHeight + rect.height)` взята оттуда же — версия из референса делила на «высота секции минус высота окна», что около нуля для секции `min-h-screen`, отсюда дёрганый, а не плавный рост), плюс отдельный `mousemove`-листенер для параллакса колец. Появление контента (силуэт/бейдж/заголовок/текст/кнопка) — через `framer-motion` `whileInView` со ступенчатыми задержками, как в `Hero.tsx`/`ClientReviews.tsx`, а не через `IntersectionObserver` + CSS-классы из исходного референса. CTA — обычный `Button` (`variant="primary"`, orange) + `ArrowCircle` (`variant="bare"`), не самодельная кнопка-пилюля.

## Вёрстка

### Ширина страницы и отступы

Каждая страница (`<main>` в `src/app/**/page.tsx`) — **всегда** `w-full mx-auto max-w-6xl px-6` снаружи, без исключений. Тот же max-width, что у навбара и у всех секций лендинга. Если контент — это чтение текста (не широкая сетка), внутри делается вложенный более узкий блок (`max-w-2xl`/`max-w-3xl`) для читаемости, но внешняя рамка страницы не меняется.

⚠️ **`w-full` здесь обязателен, не декоративный класс.** `<main>` — прямой ребёнок `<body class="flex flex-col">` в `layout.tsx`, то есть flex-элемент вертикального контейнера. `margin: auto` по горизонтали (это и есть `mx-auto`) на flex-элементе отключает его растяжение по кросс-оси (`align-items: stretch` перестаёт действовать) — без `w-full` `<main>` схлопывается до ширины контента (shrink-to-fit) вместо честных `max-w-6xl`, и `mx-auto` в этом случае центрирует уже урезанный блок, а не разницу между `max-w-6xl` и реальной шириной экрана. Баг тихий (страница просто визуально уже, чем остальные) — если новая страница выглядит "не на всю ширину" при том что классы вроде бы правильные, первым делом проверить именно это.

### Прочее

- Радиусы: карточки/крупные блоки — `rounded-2xl`/`rounded-3xl`, кнопки/бейджи/навбар — `rounded-full`.
- Секции — `py-28` вертикального отступа (просторная вёрстка в духе Wise), часто разделены `border-t border-zinc-200`.
- Анимации появления при скролле — `motion.div` с `initial={{opacity:0, y:16}}`, `whileInView`, `viewport={{once:true}}` — устоявшийся паттерн, использовать его же в новых секциях.
- **Навбар** (`src/components/Navbar.tsx`) — плавающая pill-капсула (`rounded-full`, отступ `top-4` от края экрана), стеклянный эффект (`backdrop-blur-xl`, полупрозрачный фон), очень лёгкая обводка (`border-zinc-950/[0.06]`), без тени. Справа от списка ссылок — отдельная primary-кнопка (не просто ссылка в общем списке).

## Hero-секции страниц

Каждая страница (кроме `/`) начинает контент с "hero"-блока: крупный заголовок + вводный текст, иногда с CTA. Эти блоки унифицированы:

- **Заголовок** — всегда шрифт `Unbounded` (тот же `next/font/google` подключённый локально в каждом файле, что и в `Hero.tsx` на главной), не стоковый `--font-sans`. Есть в `Hero.tsx`, `PortfolioHero.tsx`, `about/page.tsx`, `contact/page.tsx`. Логотип `SouCampus` в навбаре — тоже `Unbounded`, но остальной текст навбара (пункты меню, кнопка) — обычный `--font-sans`, это осознанное исключение только для бренд-имени.
- **Отступ от навбара — фиксированный `pt-20` (80px)** на секции/контейнере, который держит заголовок, у **всех** hero-блоков включая главную. Раньше `Hero.tsx` центрировал контент через `min-h-[85vh] + justify-center`, из-за чего отступ от навбара "плавал" в зависимости от высоты окна и не совпадал с остальными страницами ни при каком фиксированном пикселе на других страницах — сейчас все страницы используют одно и то же фиксированное значение, поэтому отступ идентичен на любом размере окна. Значение `pt-20` — ближайший стандартный шаг шкалы Tailwind к "в 1.5 раза меньше" исходного `pt-28` (112px → ~75px), выбран сознательно вместо произвольного пикселя.
- **Радиальный градиент-подсветка позади навбара** — общий приём: `absolute left-1/2 -top-32 -z-10 w-full max-w-[90rem] -translate-x-1/2` + `bg-[radial-gradient(circle_at_50%_0%,rgba(R,G,B,0.35),transparent_70%)]`, цвет акцента страницы (orange на `/`, blue на `/contact`, lime на `/portfolio`). Важно: этот div должен быть **прямым ребёнком** позиционированной (`relative`) секции, у которой сверху нет отдельного паддинг-обёртки — иначе `-top-32` не дотягивается до навбара. Паддинг-отступ заголовка (`pt-20`) вешается на ту же `relative`-секцию, а не на обёртку вокруг градиента — `top` для `position: absolute` считается от padding-края позиционированного предка независимо от значения его собственного `padding-top`, так что паддинг на этой секции никак не портит позицию градиента. `max-w-[90rem]` обязателен: `circle`-радиус по умолчанию считается по `farthest-corner` относительно ширины блока, и если растянуть его на всю `inset-x-0` ширину, на широких мониторах круг раздувается непропорционально и выглядит криво — капаем ширину, а не даём ей гнаться за viewport.
- Высота градиент-блока должна быть достаточно большой (например `h-[36rem]`+), иначе виден жёсткий срез снизу вместо плавного затухания — сравнивай с `Hero.tsx` (`h-[42rem]`), не бери маленькое значение наугад.

## Иконки

- **Phosphor Icons** (`@phosphor-icons/react`) — стандартная библиотека иконок сайта, использовать вместо самодельных SVG/текстовых символов (кроме `ArrowCircle.tsx`, у него свой устоявшийся path, не трогать). Пример — плюсик в FAQ на `/contact`.
- В серверных компонентах (страницы без `"use client"`) импортировать из **`@phosphor-icons/react/dist/ssr`**, не из корня пакета — эта сборка не требует клиентской границы. Обычный `@phosphor-icons/react` тянет клиентский контекст (вес/цвет по умолчанию через `IconContext`) и годится только внутри `"use client"`-компонентов.

## Анимация перехода между страницами — `src/components/PageTransition.tsx`

- Сетка 16 колонок × сколько влезет по высоте, клетки — честные квадраты (размер = `window.innerWidth / 16`, число рядов пересчитывается при resize).
- Три типа клеток по случайному числу: пустые (приглушённый `zinc-200`), белые (`bg-[#fbfbff]`), акцентные (оранжевые). Акцентные — не один плоский `orange-400`, а вразнобой из массива нескольких оттенков (от светлого к праймери), так волна выглядит живее.
- Включена не на всех `<Link>` сайта, а точечно — только там, где явно проставлен атрибут `data-page-transition="true"`.
- Клик перехватывается на `document` в **capture-фазе**, `preventDefault` + `stopPropagation`, реальный `router.push()` — вручную по таймеру. Без этого — гонка между анимацией и мгновенной навигацией Next.js.
- Подробности механики (волна снизу вверх, двухфазная анимация appearing/revealing) — см. комментарии в самом файле, это финальная стабильная версия, менять аккуратно.
