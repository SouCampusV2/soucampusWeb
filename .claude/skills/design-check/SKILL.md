# Design Check

Проверка текущего diff (`git diff`, плюс новые нетреканые файлы через `git status`) на соответствие правилам из `DESIGN.md`. Не спрашивать, что проверять — сам смотреть изменённые `.tsx`-файлы в `src/`.

## Порядок действий

1. Прочитать `DESIGN.md` целиком (если ещё не прочитан в этой сессии) — правила ниже кратко продублированы, но `DESIGN.md` первичен, при расхождении верить ему.
2. Посмотреть `git diff` и список новых файлов — собрать список реально изменённых/новых `.tsx`/`.css` файлов.
3. По каждому файлу пройтись по чек-листу:

### Цвета

- Используются только `orange` (primary), `blue` (secondary), `lime` (tertiary) как акцентные цвета. Никаких `emerald`, `violet`, `red`, `green`, `purple` и т.п. — если такой цвет встретился, это нарушение.
- Кнопки/CTA — **только orange или нейтральный** (secondary-кнопка без фона). `blue`/`lime` не красят кнопки — только бейджи, обводки, декор.
- Нейтральная палитра — `zinc`, не `gray`/`slate`.
- "Белый" фон — только `bg-[#fbfbff]`, никогда `bg-white`.
- Тёмный текст — `zinc-950` (заголовки), `zinc-600`/`zinc-700` (тело), `zinc-500` (второстепенный).

### Компоненты

- Все кнопки собраны через `src/components/Button.tsx` (`variant="primary"|"secondary"`, `size="sm"|"md"|"lg"`) — не собраны вручную из `<button>`/`<a>` с Tailwind-классами россыпью.
- Стрелки-кружки — только через `src/components/ArrowCircle.tsx`, не пересобранный SVG в новом месте.
- Async/ещё не готовый контент — через `src/components/Skeleton.tsx`, не пустой div и не спиннер.

### Типографика

- H1 (заголовок страницы) — `text-4xl font-extrabold tracking-tight sm:text-5xl` (кроме Hero на главной — у него свой крупный размер).
- H2 (заголовок секции) — `text-3xl font-bold tracking-tight sm:text-4xl`.
- H3 (заголовок в карточке) — `text-lg font-semibold`.
- Основной шрифт — Plus Jakarta Sans (через `--font-sans`), не новый шрифт без причины.

### Вёрстка

- `<main>` каждой страницы — `mx-auto max-w-6xl px-6`, без исключений.
- Радиусы: карточки — `rounded-2xl`/`rounded-3xl`; кнопки/бейджи/навбар — `rounded-full`.
- Вертикальный отступ секций — `py-28`.
- Анимации появления при скролле — паттерн `motion.div` с `initial={{opacity:0, y:16}}`, `whileInView`, `viewport={{once:true}}` — не выдумывать новый способ анимации без причины.
- `dark:`-классы — мёртвый код (тема всегда светлая), если встретились — можно смело вычищать по ходу.

## Формат ответа

Кратко, по пунктам — только реальные нарушения (не хвалить за то, что "всё ок" по каждому пункту чек-листа, если нарушений нет вообще — одна строка "нарушений не найдено"). На каждое нарушение:

- файл и строка;
- что не так;
- какое правило `DESIGN.md` нарушено;
- конкретная предлагаемая правка (не только "поправь", а именно что заменить на что).

Не переходить к исправлению файлов без подтверждения пользователя — только отчёт.
