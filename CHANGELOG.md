(2) `/portfolio/[slug]` рендерится динамически (`ƒ` в выводе build), а не статически: чтение `searchParams` (`?from=home` для back-ссылки) выключает SSG у всей страницы — починить, заменив на клиентское чтение (`useSearchParams` в маленьком клиентском компоненте back-ссылки под `<Suspense>`). Устаревшие места в `STRUCTURE.md` (шапка "не реализовано", схема stats с color/bar) и `CLAUDE.md` (п.12 про Этап 3) — исправлены.
- **Этап 4, подэтап A — каталог магазина, сделано.** План магазина принят: A каталог → B покупка одного товара (Stripe Checkout + вебхук + signed URL из приватного Storage) → C корзина (localStorage) + мульти-чекаут → D Supabase Auth + профиль "My purchases" (гостевой чекаут остаётся) → E своя админка (до неё — Table Editor, как в Этапе 3). Магазин стартует в текущем роутинге (`(site)/shop`), поддомен — позже через Vercel domain mapping, без перестройки кода. Сделано в A: миграция `20260722120000_products.sql` (таблица `products`: `price_cents` integer — деньги никогда не float, сразу в формате Stripe; `file_path` nullable до подэтапа B; `is_published` default false — черновики невидимы; RLS `for select using (is_published)` — неопубликованное не отдаст даже прямой запрос к REST API с anon-ключом, в отличие от `projects` с `using (true)`); `src/lib/products.ts` — тот же паттерн границы, что projects.ts (`ProductRow` → `rowToProduct()` → `Product`), + тест на маппинг и на `Number()` для price_cents; страницы `/shop` (каталог с hero по общему паттерну, пустое состояние) и `/shop/[slug]` (деталка по образцу работы, кнопка Buy пока ведёт в Discord — Stripe в B); noindex у `/shop` стал **условным** (`generateMetadata`: пока нет опубликованных товаров — noindex, с первым товаром страница сама открывается поисковику без правки кода), та же логика в `sitemap.ts` (товары и `/shop` попадают в карту только когда есть опубликованные); `isTrackablePath` в `views.ts` расширен на `/shop/[slug]` — счётчик просмотров считает и товары. Lint/тесты (42) /build — зелёные, таблица создана в Supabase через SQL Editor.
- **Вопрос "а вдруг маркетплейс на 1000 товаров и продавцов" разобран:** стек не меняется (1000 строк для Postgres — ничто), продавцы = Stripe Connect + RLS "видишь только своё" + модерация, микросервисы не проектируются заранее — единственный реальный будущий кандидат на вынос — worker обработки файлов карт (валидация/превью, возможно на Java из-за Minecraft-тулинга).
