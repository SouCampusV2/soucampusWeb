# Мобильная/планшетная адаптация — рабочий план

Рабочий документ на время responsive-прохода. Как только правила устаканятся — перенести финальную версию в `DESIGN.md` (там уже есть заготовка "зафиксировать брейкпоинты/паттерны" в CLAUDE.md), а этот файл можно будет удалить.

## Мини-система брейкпоинтов (предложение)

Сейчас на сайте фактически используются только `sm:` (640px) и местами `lg:` (1024px) — `md:` (768px, классический планшет) не используется нигде ни разу. Из-за этого весь диапазон 640–1023px просто наследует мобильную-или-почти-десктопную вёрстку без какой-либо собственной настройки.

Предлагаемая роль каждого брейкпоинта на этом сайте:

- **база (без префикса, < 640px)** — телефон. Всё в один столбец, полноразмерные touch-цели, никаких hover-only взаимодействий.
- **`sm:` (640px)** — большой телефон/маленький планшет. Обычно первая точка, где что-то может стать 2 колонки вместо 1.
- **`md:` (768px)** — планшет. **Новая точка, которую сейчас нигде не используем.** Сюда — то, что на телефоне 1 колонка, на десктопе 3+, но резкий скачок 1→3 на iPad выглядит криво: даём 2 колонки именно здесь.
- **`lg:` (1024px)** — десктоп. Сложные bento-сетки, многоколоночные layout, всё что не нужно телефону/планшету вообще.

Правило по умолчанию при добавлении новой секции: если сетка идёт `grid-cols-1` → `lg:grid-cols-N` (N≥3) без `sm:`/`md:` шага между ними — это подозрительно, планшет получает то же самое, что телефон, при вдвое большей ширине. Добавлять `sm:`/`md:` промежуточный шаг, если контент это позволяет.

## Реальные баги (не просто "не оптимизировано для планшета", а видимо ломается)

1. **`ClientReviews.tsx`** — стрелки карусели `hidden … sm:flex` (скрыты на телефоне), при этом карточка `CARD_WIDTH = 340` px хардкод. На 375px экране это почти вся ширина + нет свайпа + нет стрелок → на телефоне физически нельзя пролистать отзывы дальше первого.
2. **`PortfolioHero.tsx`** — раскрытие панели карусели построено на `onMouseEnter`/`onMouseLeave` (только hover), на тач-устройствах эффекта нет вообще, есть только `onClick` (сразу переход). Плюс стрелки тоже `sm:flex` (скрыты на телефоне).
3. **`/portfolio/[slug]` галерея** — `grid-cols-2` без адаптации ни под один брейкпоинт → на телефоне два тонких (~155px) столбца фото, у всех остальных 2-кол сеток на сайте есть `sm:`-фоллбек, тут его нет.
4. **`AuthorCta.tsx` кольца** — все размеры (`size`, `riseUp`) — фиксированные px (920–1920px), никак не завязаны на ширину экрана. На телефоне (375px) кольца в разы больше вьюпорта в абсолютных пикселях — то есть визуально будут смотреться так же нелепо крупно, как на десктопе 1920px они смотрятся нормально. Плюс рост при скролле (`growTo` до 1.55×) поверх этого.
5. **`-mt-[82px]` в `AuthorCta.tsx`** — компенсирует высоту навбара, которая сейчас одинакова на всех брейкпоинтах (лого + один гамбургер), так что число стабильно; но это хардкод без `sm:`-варианта — если высота навбара когда-то изменится на мобильном, придётся не забыть поправить и это.

## Не баги, но "планшет = телефон" (второй приоритет)

- `PricingSection.tsx` — сетка только `grid-cols-1` → `lg:grid-cols-6`, нет ничего между. На iPad (768px) один в один как на телефоне.
- `HowItWorks.tsx` — `sm:grid-cols-2 lg:grid-cols-5` на 5 карточек — при 2 колонках последняя карточка сиротой.
- `CountriesMarquee.tsx` — фикс `128px` кружки, без разницы по брейкпоинтам (не ломается, просто на телефоне видно меньше кружков за раз).
- `Navbar.tsx` — переключение гамбургер↔полное меню происходит на `sm:` (640px), не проверено, влезает ли весь список ссылок + кнопка ровно на 640-767px без обжатия.

## Сделано (первый проход)

- **`Navbar.tsx`** — мобильное меню: (1) убран баг с `rounded-full` на выросшем блоке (превращался в блоб) — теперь дропдаун `position: absolute`, не толкает контент страницы и не меняет форму пилюли; (2) меню закрывается по смене `pathname` (не по клику — клик перехватывает `PageTransition` в capture-фазе и глотает всплытие); (3) непрозрачность фона одинаковая что в открытом, что в закрытом состоянии.
- **`ClientReviews.tsx`** — карточка теперь `w-[78vw] sm:w-[340px]` (не фиксированный px), ширина/шаг анимации и `visible`-count измеряются в рантайме через `ResizeObserver`-подобный resize-listener; стрелки видны на всех брейкпоинтах; добавлен `drag="x"` — карусель свайпается.
- **`PortfolioHero.tsx`** — ниже `sm:` пятипанельный hover-ряд заменён на обычную одну-картинку-за-раз карусель (свайп + точки-индикатор), стрелки видны всегда; на десктопе первый тап по неактивной панели теперь активирует превью вместо мгновенного перехода (второй тап переходит).
- **`/portfolio/[slug]`** — галерея `grid-cols-2` → `grid-cols-1 sm:grid-cols-2`.
- **`AuthorCta.tsx`** — кольца уменьшены на мобильном через `scale-50 sm:scale-100` на отдельной (не управляемой JS) обёртке — не конфликтует с инлайн-`transform` от scroll/parallax анимации.
- **`PortfolioHero.tsx`** гродиент-фон — был `w-full` внутри секции, вложенной в padded `<main>` → на мобильном не доставал до краёв экрана; заменён на `w-screen`.
- **`CountriesMarquee.tsx`** — кружки флагов `h-32 w-32` → `h-20 w-20 sm:h-32 sm:w-32`.
- **`PageTransition.tsx`** — число колонок анимации перехода теперь `8` на `< 640px` (было фиксированных `16`), клетки крупнее на телефоне.
- **`/about`** — гэпы между hero-секцией/таймлайном/заключением/фото уменьшены на мобильном (`mt-24`→`mt-12 sm:mt-24` и т.п.).

Проверено скриншотами на 375/390/430/768px (Chrome headless) — навбар не прыгает, обе карусели свайпаются и листаются кнопками, меню закрывается при переходе, кольца не разрывают экран.
